## 摘要
网络流量分类能够实现网站指纹识别、入侵检测和服务质量管理。然而，在现实部署条件下开发能够捕获稳定且可泛化的流量模式的方法仍然具有挑战性。我们提出FlowCLIP，一个对比预训练框架，仅使用侧信道特征（数据包到达间隔时间、数据包大小和数据包方向）从加密流量中预测域名。FlowCLIP通过CLIP风格的对比目标将流量流表示与域名表示对齐，从而将原始域名作为文本监督。预训练的流量编码器随后被冻结，并通过在规范化域名标签上进行线性探测来评估。我们在一个大规模QUIC流量数据集上使用基于时间跨度的协议评估FlowCLIP，其中模型在第1周的流量上训练，并在第2-4周的流量上评估。FlowCLIP在后续评估周中优于有竞争力的机器学习基线，表明原始域名为学习可迁移的加密流量表示提供了文本监督信号。

## Abstract
Network traffic classification enables website fingerprinting, intrusion detection, and Quality of Service management. However, developing methods that capture stable and generalizable traffic patterns under realistic deployment conditions remains challenging. We introduce FlowCLIP, a contrastive pretraining framework for domain name prediction from encrypted traffic using only side-channel features: packet inter-arrival times, packet sizes, and packet directions. FlowCLIP uses raw domain names as textual supervision by aligning traffic flow representations with domain name representations through a CLIP-style contrastive objective. The pretrained traffic encoder is then frozen and evaluated through linear probing on canonicalized domain name labels. We evaluate FlowCLIP on a large-scale QUIC traffic dataset using a time-based protocol, where models are trained on Week 1 traffic and evaluated on traffic from Weeks 2-4. FlowCLIP outperforms competitive machine learning baselines across later evaluation weeks, suggesting that raw domain names provide a textual supervision signal for learning transferable encrypted traffic representations.

## 1. 论文的核心问题与整体含义（研究动机和背景）
加密流量分类在真实部署中面临三大挑战：
- **特征表示**：许多方法依赖IP地址、原始字节等可能泄露流量特征的信号（如SNI、负载），这些在加密环境下不可用或者泛化性差；模型容易学习到数据集特有伪影而非通用模式。
- **评估设置**：常见做法使用随机包级拆分导致标签泄漏（同一流的数据出现在训练和测试集），或者实验室环境下的数据集缺乏时间漂移，无法反映真实部署中的概念漂移。
- **应用标签**：域名SNI被广泛用于标注，但缺乏标准化的映射规则，导致不同数据集的类别粒度不一致，引入人工偏差。

FlowCLIP的动机是**利用原始域名作为文本监督信号**，通过对比预训练学习时间可迁移的加密流量表示，仅依赖包间间隔、包大小和包方向这些侧信道特征，而无需解密流量或依赖可能被隐藏的标识符。

## 2. 论文提出的方法论

### 核心思想
采用CLIP风格的对比学习，将双向流量流的表示与对应原始域名的表示在共享嵌入空间中对齐。预训练后冻结流量编码器，线性分类头在规范化域名标签上进行预测。

### 关键技术细节
- **输入特征**：每个流取前30个包，使用包大小、包方向（通过有符号大小编码）、方向感知的包间间隔时间（计算自同方向上一包的时间差，去除交叉方向干扰）。不足部分零填充。
- **域名编码**：将原始域名按层次分解为组件（从TLD到子域，例如`mail.google.com` → `[com, google, mail]`），每个组件映射到可学习嵌入，加上位置嵌入，再对所有非填充组件做平均池化。
- **流量编码器**：Transformer编码器（4层，4注意力头，d_model=256），带填充掩码，对非填充位置做平均池化得到流级表示。
- **预训练**：使用对称的对比损失（flow→domain 和 domain→flow），温度参数τ=0.07，批量大小1024，学习率3e-4，权重衰减0.01，训练50轮（早停patience=10）。
- **标签规范化**：基于第1周训练数据自动构建标签空间。分解所有域名组件，保留出现≥2次的组件，替换低频组件为`<unk>`；保留覆盖99%流量的最频繁规范化域名，其余归入“other”类，最终得到1915个类别（含“other”）。第2-4周沿用第1周的词汇表和类别集合。

## 6. 论文的主要结论与发现
- FlowCLIP在所有测试周上准确率和加权F1均显著优于1-NN和XGBoost（例如第2周：准确率81.38% vs 71.17%和50.17%）。
- 所有模型的性能从第2周到第4周均下降，表明时间动态变化仍然是一个挑战。
- 宏F1明显低于加权F1，说明稀有类别分类困难。
- 消融表明：对比预训练相比直接监督训练能够提升常见类别的性能（准确率和加权F1更高），但监督Transformer在宏F1上更好（对稀有类更友好）；MLP头相比线性头能更好地利用预训练表示。
- 原始域名文本可以作为有效的监督信号学习可迁移的加密流量表示，无需人工定义类别。

## 7. 优点
- **方法创新**：首次利用原始域名作为文本监督进行对比预训练，避免了人工标签偏差；域名编码保留了层次结构信息，比扁平类别更丰富。
- **评估真实**：采用时间基评估（周跨度），反映部署中可能出现的分布漂移；流级别拆分避免泄漏。
- **特征安全**：仅使用包间间隔、大小和方向这些始终可观测的侧信道特征，不依赖可能被加密或隐藏的标识符（如SNI、IP）。
- **自动化标签**：提出基于频率的域名规范化流程，减少人工干预可重复。
- **迁移潜力**：预训练后的编码器可适配不同标签空间（通过更换分类头）。
- **开源友好**：使用了公开大规模真实QUIC数据集。

## 8. 不足与局限
- **数据集单一**：仅针对QUIC流量，未评估TCP/TLS或其他协议；QUIC可能具有特殊模式，结论通用性存疑。
- **标签处理不完善**：频率过滤虽自动化，但未语义合并实例特定组件（如shard ID），导致“other”类可能过大；规范化后的标签仍保留长尾分布。
- **静态预训练不抗漂移**：预训练只基于第1周，之后性能下降，缺乏在线适应或持续学习。
- **稀有类别表现差**：宏F1仅约40-50%，对实际应用（如安全检测）中罕见但关键的服务可能不够用。
- **基线选择不够强**：未与当前最优深度模型或自监督对比学习模型（如SimCLR变体）对比，XGBoost在本设定下表现极差（第1轮即早停），可能已非公平比较。
- **域名可用性受威胁**：若未来ECH广泛部署则SNI不可见，FlowCLIP将无法获得文本监督；论文未讨论此场景下的替代方案。
- **推理效率**：虽比1-NN快，但比XGBoost慢约4倍（0.030 vs 0.007ms），且依赖GPU。
