## 摘要
现代网络入侵检测系统（NIDS）陷入了一个结构性矛盾：承载最高威胁情报的协议恰恰是那些经过TLS 1.3和QUIC加密的协议，其负载检测毫无结果。我们提出了一个更简单的问题——如果攻击签名不在字节中，而在节奏中呢？——并通过将网络流视为一种语法完全由L3/L4数据包元数据（长度、到达间隔时间、TTL、TCP标志以及哈希端口号）书写的语言来回答它。我们提出了PLM-NIDS，该模型依次证明了三个主张。（1）语法存在且可学习：一个在344,232条无标签的周一流量数据上训练的RWKV-4状态空间模型实现了0.204的因果语言模型验证损失，表明良性流量具有可预测且统计上一致的结构。（2）攻击违背了这种语法：在训练时未使用任何攻击标签的情况下，基于每个流的困惑度得分以PR-AUC = 0.93清晰地分离了良性流和攻击流。（3）这种分离在架构上是不平凡的：一个在相同令牌序列上训练的LSTM退化为多数类预测器（ROC-AUC约为0.50，通过始终预测“攻击”获得F1 = 0.91），证明了RWKV的因果预训练提供了直接分类器无法获得的归纳偏置。监督微调进一步将PR-AUC提升至0.94，ROC-AUC提升至0.75，在校准操作阈值下精确率达到97.7%。RWKV骨干网络的O(T)循环推理实现了无需流缓冲的逐包流式处理，使得PLM-NIDS在线速下操作可行。由于它仅读取IP/TCP/UDP头部，因此本质上与加密无关：TLS 1.3、QUIC以及未来的加密协议都能被透明处理。

## Abstract
Modern network intrusion detection systems (NIDS) are caught in a structural contradiction: the protocols carrying the highest threat intelligence are precisely those encrypted under TLS 1.3 and QUIC, where payload inspection yields nothing. We ask a simpler question -- what if the attack signature is not in the bytes, but in the rhythm? -- and answer it by treating network flows as a language whose grammar is written entirely in L3/L4 packet metadata: length, inter-arrival time, TTL, TCP flags, and hashed port numbers. We present PLM-NIDS, which proves three claims in sequence. (1) The grammar exists and is learnable: a RWKV-4 state-space model trained on 344,232 unlabelled Monday flows achieves a causal LM validation loss of 0.204, demonstrating that benign traffic has predictable, statistically consistent structure. (2) Attacks violate this grammar: the per-flow perplexity score cleanly separates benign from attack flows with PR-AUC = 0.93 using zero attack labels at training time. (3) This separation is architecturally nontrivial: an LSTM trained on identical token sequences degenerates to a majority-class predictor (ROC-AUC approximately 0.50, F1 = 0.91 by always predicting "attack"), proving that RWKV's causal pre-training provides an inductive bias unavailable to direct classifiers. Supervised fine-tuning further raises PR-AUC to 0.94 and ROC-AUC to 0.75, with a precision of 97.7% at the calibrated operating threshold. The RWKV backbone's O(T) recurrent inference enables per-packet streaming without flow buffering, making PLM-NIDS operationally viable at line rate. Because it reads only IP/TCP/UDP headers, it is inherently encryption-agnostic: TLS 1.3, QUIC, and future encrypted protocols are handled transparently.

### 论文提出的方法论

- **核心思想**：用语言模型建模正常网络流的元数据模式，通过困惑度（perplexity）检测异常。
- **关键技术细节**：
    1. **DPI-free Tokenization**：每个包转为9个token，包括方向、对数分箱长度/间隔/TTL、L4协议、哈希源/目的端口（取模）、TCP标志。词汇表仅227个token，不读取任何负载字节。
    2. **模型架构**：使用RWKV-4状态空间模型（6层，隐藏维256，约480万参数）。RWKV的WKV操作具有线性复杂度O(T)和O(1)流式状态，支持逐包处理。
    3. **两阶段训练策略**：
        - Phase 1（无监督）：仅在周一良性流量（344,232流）上训练因果语言模型（最小化交叉熵），学到正常语法的概率分布。
        - Phase 2（监督微调）：在全部标记数据上联合优化LM损失和分类损失，攻击类权重为8，前两个epoch冻结骨干保护表示。
    4. **推理流水线**：包到达→ 5元组查流状态 → 每包9个token → RWKV step()处理 → 累加负对数似然 → 若困惑度超门限则告警。无需缓存完整流。
- **评分模式**：
    - PLM-PPL（无监督）：仅用困惑度。
    - PLM-CLS（监督）：分类头输出。
    - PLM-CMB（组合）：几何平均（实际表现更差）。

### 论文的主要结论与发现

1. **良性语法可学**：Phase 1在无标签良性数据上达到val loss=0.204，证明正常流量有可预测结构。
2. **攻击违反语法**：无监督PPL得分的PR-AUC=0.93，精度97.7%，确认攻击流在困惑度上偏离正常。
3. **架构必要**：同一token下LSTM退化为多数类预测器（ROC-AUC≈0.50，F1来自“永远预测攻击”），RWKV的因果预训练提供了防止退化的归纳偏置。
4. **监督微调提升**：PLM-CLS达到PR-AUC=0.94、ROC-AUC=0.75、精确率97.7%，假阳率5%（按设计）。
5. **O(T)流式推理**：RWKV支持逐包处理，运算量比Transformer节省521倍（95百分位长度），适用于线速部署。
6. **加密无关**：仅读L3/L4头部，透明处理TLS 1.3/QUIC。

### 优点

- **完全DPI-free**：仅依赖L3/L4头部元数据，不读取负载，法律合规且加密兼容。
- **创新的问题框架**：将入侵检测转化为语言模型异常检测，将攻击特征定义为“语法违背”，而非直接学习攻击模式。
- **流式推理**：RWKV的O(T)递归与O(1)状态，无需流缓存，适合实时线速。
- **两阶段设计强调“先学正常”**：通过预训练建立正常分布，微调仅学习偏离，避免类别不平衡导致的退化——与LSTM对比实验有力支撑。
- **高精度操作点**：95% FPR TP时精度97.7%，对实际运维友好。
- **重复性和开放代码**：作者公开代码、tokenizer、评估管道。

### 不足与局限

- **单数据集评估**：仅使用CIC-IDS-2017，缺乏跨数据集（如真实企业流量、HIKARI-2021纯加密数据集）泛化验证，削弱通用性声明。
- **标签噪声**：使用天级别二分类（周一良性，其他攻击），但攻击天内部也有良性流量，被错误标记为攻击，导致召回被低估（PLM-CLS召回仅42%）。
- **低检测率的攻击类型**：暴力破解（FTP/SSH）仅9% TPR，因为低速率试探模仿正常认证元数据节奏。
- **Python级WKV效率**：当前纯Python循环处理约4,500流/分钟，远未达到10Gbps线速；作者承认需CUDA融合核才能实用。

- **模型大小与搜索**：未对embedding维、层数等进行系统搜索；RWKV v5/v6等更新架构未使用。
- **隐私与安全**：虽然不读负载，但元数据（如端口哈希）仍可能泄漏服务类型；某些攻击（如针对TLS的侧信道）作者未讨论。
